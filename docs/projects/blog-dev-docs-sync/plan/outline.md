# Git 커밋 → 개발 문서 자동 최신화 — 아웃라인

## 도입
- 개발 문서가 코드와 동기화되지 않는 문제
- "문서 업데이트 해야지" → 영원히 안 함

## 본문

### 1. 워크플로 설계
- CLAUDE.md에 명령어 정의
- 서브모듈 커밋 조회 → scope 분류 규칙
- 프로젝트별 dev/*_log.md 자동 매핑

### 2. 실제 동작 과정
- 명령어 입력부터 커밋까지 전 과정 시연
- 자동 생성되는 문서 예시 (frontend_log.md)

### 3. 코드 리뷰 문서 자동화
- 변경 파일 분석 → review.md 업데이트
- 리뷰 포인트 자동 추출

## 결론
- 문서가 코드와 항상 동기화되는 상태 유지
- 개발자는 코드에만 집중, 문서는 Claude가 처리
