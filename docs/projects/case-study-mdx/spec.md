## 목표

현재 아코디언 카드 형태의 케이스 스터디를 MDX 기반 독립 페이지로 전환하여, 각 케이스가 풍부한 콘텐츠(이미지, 코드 블록, 다이어그램)를 담을 수 있는 블로그 형태로 만든다.

## 현재 문제

1. **콘텐츠 한계** — 아코디언 안에 `<p>` 텍스트만 가능. 이미지, 코드 스니펫, 다이어그램 불가
2. **공유 불가** — 개별 URL이 없어서 특정 케이스를 링크로 공유할 수 없음
3. **SEO 불가** — 모든 케이스가 같은 페이지에 있어 검색엔진이 개별 콘텐츠로 인식 못함
4. **가독성** — 긴 텍스트가 단락 구분 없이 한 덩어리로 표시

## 변경 후 구조

```
현재:  / (page.tsx 내 아코디언)
       └── 케이스 스터디 탭 → 3개 아코디언 카드

변경:  /case-study              → 목록 페이지 (카드 리스트)
       /case-study/[slug]       → 상세 페이지 (MDX 렌더링)
```

## 범위

### 포함
- MDX 파일 기반 케이스 스터디 콘텐츠 (기존 3개 이관)
- `/case-study` 목록 페이지
- `/case-study/[slug]` 상세 페이지
- MDX 파싱 및 렌더링 설정
- 사이드바 네비게이션에 케이스 스터디 링크 연결
- 각 상세 페이지의 SEO 메타태그 (title, description, OG)

### 제외
- 기술 블로그 (`/blog`) — 후속 프로젝트에서 같은 구조 재사용
- 댓글 시스템
- 태그 필터링, 검색
- RSS 피드

## 수정/생성 대상 파일

| 파일 | 변경 내용 |
|------|----------|
| `content/case-study/*.mdx` | 케이스 스터디 MDX 콘텐츠 3개 (신규) |
| `src/app/case-study/page.tsx` | 목록 페이지 (신규) |
| `src/app/case-study/[slug]/page.tsx` | 상세 페이지 (신규) |
| `src/lib/mdx.ts` | MDX 파싱 유틸리티 (신규) |
| `src/app/page.tsx` | 케이스 스터디 탭/데이터/컴포넌트 제거, 사이드바 링크 변경 |
| `next.config.ts` | MDX 관련 설정 (필요시) |
| `package.json` | MDX 관련 의존성 추가 |

## 기술 제약

- MDX 파싱: `next-mdx-remote` 또는 `@next/mdx`
- Frontmatter: `gray-matter`
- 코드 하이라이팅: `rehype-pretty-code` 또는 `shiki`
- 상세 페이지는 서버 컴포넌트 (SSG) — SEO 확보
- 기존 사이드바 레이아웃과 일관된 디자인 유지

## 후속 확장성

이 프로젝트에서 만드는 MDX 인프라(`lib/mdx.ts`, 라우팅 패턴, 레이아웃)를 기술 블로그(`/blog/[slug]`)에서 그대로 재사용할 수 있도록 범용적으로 설계한다.
