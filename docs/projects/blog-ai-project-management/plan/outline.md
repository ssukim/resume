# 기획→디자인→개발→QA, AI로 프로젝트 관리하기 — 아웃라인

## 도입
- 소규모 팀의 프로젝트 관리 현실: 도구는 많지만 프로세스가 없다
- AI를 "팀원"으로 활용하는 접근법

## 본문

### 1. 문서 허브 설계 철학
- 왜 Notion/Jira가 아닌 Git 기반 마크다운인가
- 코드와 문서가 같은 저장소에 있는 이점

### 2. 프로세스 흐름 상세
- 미팅 → plan/scenario.md (시나리오 기반 기획)
- design/handoff.md → Figma MCP 연동
- dev/spec.md → 개발 시작점
- qa/checklist.md → 자동 QA
- history/ → 아카이빙

### 3. 피처 자동 생성
- 미팅 문서 하나로 전체 프로젝트 폴더/문서 생성
- 템플릿 시스템의 역할

### 4. 실제 사례: auto-response 프로젝트
- 기획부터 QA까지 전 과정 타임라인
- 각 단계에서 Claude가 생성한 실제 문서

## 결론
- AI는 PM을 대체하지 않지만, PM의 반복 작업을 제거
- 프로세스가 문서화되면 AI가 실행할 수 있다
