# 블로그 시리즈 아웃라인

## #2 MCP로 Claude를 슈퍼 도구로 만들기

### 도입
- MCP(Model Context Protocol)란 무엇인가
- Claude Code가 외부 도구와 연결되면 무엇이 달라지는가

### 본문
1. **Figma MCP — 디자인과 코드의 벽 허물기**
   - 설정 방법 (mcp.md 기반)
   - 실제 사용 사례: design/handoff.md 자동 생성
   - Before/After 비교
2. **Atlassian MCP — Jira/Confluence 통합**
   - Confluence 문서 검색으로 컨텍스트 확보
   - Jira 이슈 자동 생성/업데이트
3. **Playwright MCP — 브라우저를 손에 쥐다**
   - QA 자동화 (체크리스트 기반)
   - 블로그 자동 발행
   - 스크린샷 촬영

### 결론
- MCP는 Claude를 "대화 도구"에서 "실행 도구"로 전환
- 설정 한 번으로 팀 전체의 워크플로가 바뀜

## #3 Git 커밋 → 개발 문서 자동 최신화

### 도입
- 개발 문서가 코드와 동기화되지 않는 문제
- "문서 업데이트 해야지" → 영원히 안 함

### 본문
1. **워크플로 설계**
   - CLAUDE.md에 명령어 정의
   - 서브모듈 커밋 조회 → scope 분류 규칙
   - 프로젝트별 dev/*_log.md 자동 매핑
2. **실제 동작 과정**
   - 명령어 입력부터 커밋까지 전 과정 시연
   - 자동 생성되는 문서 예시 (frontend_log.md)
3. **코드 리뷰 문서 자동화**
   - 변경 파일 분석 → review.md 업데이트
   - 리뷰 포인트 자동 추출

### 결론
- 문서가 코드와 항상 동기화되는 상태 유지
- 개발자는 코드에만 집중, 문서는 Claude가 처리

## #4 Claude Code로 QA 자동화 구축하기

### 도입
- 수동 QA의 한계: 반복적, 누락 발생, 시간 소모
- AI + 브라우저 자동화로 해결

### 본문
1. **QA 체크리스트 작성법**
   - qa/checklist.md 구조
   - 시나리오 기반 테스트 항목 정의
2. **Playwright MCP 연동**
   - DEV 환경 접속 → 시나리오 자동 실행
   - 스크린샷 자동 촬영 및 저장
3. **실제 QA 실행 과정**
   - auto-response 프로젝트 QA 사례
   - 버그 발견 → 분류 → 목록 자동 업데이트
   - 결과 문서 Before/After

### 결론
- QA 시간 대폭 절감
- 재현 가능한 테스트 (스크린샷 기록)
- 개발자가 QA 부담 없이 기능에 집중

## #5 기획→디자인→개발→QA, AI로 프로젝트 관리하기

### 도입
- 소규모 팀의 프로젝트 관리 현실: 도구는 많지만 프로세스가 없다
- AI를 "팀원"으로 활용하는 접근법

### 본문
1. **문서 허브 설계 철학**
   - 왜 Notion/Jira가 아닌 Git 기반 마크다운인가
   - 코드와 문서가 같은 저장소에 있는 이점
2. **프로세스 흐름 상세**
   - 미팅 → plan/scenario.md (시나리오 기반 기획)
   - design/handoff.md → Figma MCP 연동
   - dev/spec.md → 개발 시작점
   - qa/checklist.md → 자동 QA
   - history/ → 아카이빙
3. **피처 자동 생성**
   - 미팅 문서 하나로 전체 프로젝트 폴더/문서 생성
   - 템플릿 시스템의 역할
4. **실제 사례: auto-response 프로젝트**
   - 기획부터 QA까지 전 과정 타임라인
   - 각 단계에서 Claude가 생성한 실제 문서

### 결론
- AI는 PM을 대체하지 않지만, PM의 반복 작업을 제거
- 프로세스가 문서화되면 AI가 실행할 수 있다

## #6 Claude Code로 블로그 자동 발행 시스템 만들기

### 도입
- 블로그 글은 쓰고 싶지만 발행이 번거롭다
- 작성부터 발행까지 전 과정 자동화

### 본문
1. **워크플로 설계**
   - CLAUDE.md의 "블로그 써줘" 명령어 구조
   - 주제 선정 → 톤 확인 → 메타 수집 → 작성 → 발행
2. **Playwright MCP로 폼 자동화**
   - DEV 환경 블로그 작성 페이지 접속
   - 제목, 내용, 태그, 썸네일 자동 입력
   - 미리보기 확인 후 발행
3. **실제 발행 시연**
   - 팀 블로그 실제 포스트 발행 과정
   - 스크린샷과 함께 단계별 설명

### 결론
- 블로그 발행 소요 시간: 1시간 → 10분
- 컨텐츠 품질에만 집중, 나머지는 AI가 처리
