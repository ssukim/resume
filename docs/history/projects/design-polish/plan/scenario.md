## 배경

- shadcn/ui 기본 다크 테마를 그대로 사용해 다른 개발자 이력서와 차별점이 없음
- 채용 담당자가 하루에 수십 개 이력서를 보는 상황에서, 시각적으로 기억에 남지 않으면 다시 찾아오지 않음
- 콘텐츠 품질은 높지만 비주얼이 뒷받침하지 못하고 있음

## 사용자 스토리

### 스토리 1: 첫 인상에서 전문성 느끼기
- **사용자**: 채용 담당자
- **행동**: 사이트에 처음 접속한다
- **결과**: 색상과 레이아웃에서 정돈된 인상을 받고, 프로필 영역에서 누구인지 즉시 파악한다
- **우선순위**: P0 (필수)

### 스토리 2: 긴 이력서를 편하게 읽기
- **사용자**: 채용 담당자
- **행동**: 프로젝트 섹션을 스크롤하며 읽는다
- **결과**: 섹션 간 구분이 명확하고, 텍스트 가독성이 좋아 피로감 없이 읽을 수 있다
- **우선순위**: P0 (필수)

### 스토리 3: 케이스 스터디 탐색
- **사용자**: 채용 담당자
- **행동**: 이력서에서 케이스 스터디 탭으로 전환하고, 아코디언을 열어 상세 내용을 본다
- **결과**: 탭 전환과 아코디언 열림이 부드럽게 동작해 완성도 높은 사이트라고 느낀다
- **우선순위**: P1 (중요)

### 스토리 4: 모바일에서 사이드바 탐색
- **사용자**: 채용 담당자 (모바일)
- **행동**: 사이드바를 열어 프로필과 연락처를 확인한다
- **결과**: 프로필 영역이 깔끔하고 연락처가 바로 보인다
- **우선순위**: P1 (중요)

## 성공 지표

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 시각적 차별화 | 무채색 탈피, 액센트 컬러 1개 이상 적용 | 수동 확인 |
| 가독성 | 다크모드 본문 읽기 편함 | 수동 확인 |
| 인터랙션 | 탭 전환, 아코디언에 트랜지션 적용 | 수동 확인 |
| 프로필 완성도 | 아바타 + 이름/타이틀 위계 명확 | 수동 확인 |
