## 배경

- 현재 단일 컬럼 + 상단 탭 구조는 일반적인 블로그 형태로, 포트폴리오 사이트로서 차별화가 부족함
- 사이드바 레이아웃으로 전환하면 프로필 정보가 항상 노출되어 방문자가 "누구의 이력서인지" 즉시 파악 가능
- 데스크톱에서 사이드바와 콘텐츠가 독립 스크롤되어, 긴 이력서를 읽는 동안에도 프로필/네비게이션에 접근 가능

## 사용자 스토리

### 스토리 1: 데스크톱에서 이력서 탐색
- **사용자**: 채용 담당자
- **행동**: 사이트에 접속해 이력서와 케이스 스터디를 탐색한다
- **결과**: 사이드바에서 프로필과 연락처를 항상 확인할 수 있고, 탭 전환으로 콘텐츠를 빠르게 전환할 수 있다
- **우선순위**: P0 (필수)

### 스토리 2: 모바일에서 이력서 탐색
- **사용자**: 채용 담당자 (모바일)
- **행동**: 모바일에서 사이트에 접속해 이력서를 확인한다
- **결과**: 상단 바에서 이름을 확인하고, 햄버거 메뉴로 네비게이션과 연락처에 접근할 수 있다
- **우선순위**: P0 (필수)

### 스토리 3: 탭 전환 시 스크롤 초기화
- **사용자**: 방문자
- **행동**: 이력서를 스크롤하다가 케이스 스터디 탭으로 전환한다
- **결과**: 메인 콘텐츠 영역이 상단으로 초기화되어, 이전 스크롤 위치가 남아있지 않다
- **우선순위**: P1 (중요)

### 스토리 4: 사이드바에서 핵심 역량 확인
- **사용자**: 채용 담당자
- **행동**: 사이드바를 통해 지원자의 핵심 기술을 빠르게 파악한다
- **결과**: Core Skills 영역에서 주요 기술 4개를 Badge로 즉시 확인할 수 있다
- **우선순위**: P2 (개선)

## 성공 지표

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 데스크톱 사이드바 정상 동작 | 사이드바 고정, 메인 독립 스크롤 | 수동 확인 (1024px+) |
| 모바일 반응형 정상 동작 | 햄버거 메뉴 슬라이드인/아웃 | 수동 확인 (<1024px) |
| 기존 콘텐츠 깨짐 없음 | 모든 섹션 정상 렌더링 | 수동 확인 |
| 탭 전환 스크롤 초기화 | 전환 시 상단으로 이동 | 수동 확인 |
